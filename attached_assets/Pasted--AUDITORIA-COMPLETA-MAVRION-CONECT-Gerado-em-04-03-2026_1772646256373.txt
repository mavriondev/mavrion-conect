🔴 BUG 1 — camposEspecificos não existe no banco de dados
   IMPACTO: CRÍTICO — silencioso e perigoso

   O código em ativo-detalhe.tsx salva dados em `camposEspecificos`:
     - Checklist de due diligence
     - Dados Embrapa (solo, NDVI, clima)
     - Coordenadas GPS (quando implementadas)

   Mas o schema.ts não tem a coluna `camposEspecificos` na tabela assets.
   O Drizzle ignora silenciosamente campos desconhecidos no update().
   Resultado: todos esses dados somem ao recarregar a página.

   Para a apresentação: ao clicar num checkbox da due diligence e recarregar,
   o checkbox vai desaparecer. Isso é fatal numa demo ao vivo.

   A tabela tem `attributesJson` mas o código usa `camposEspecificos`.
   Solução: renomear/mapear ou adicionar a coluna.

🔴 BUG 2 — Matching Engine: investidor vazio pontua quase igual a match perfeito
   IMPACTO: CRÍTICO para credibilidade da demo

   Código atual:
     if (investorTypes.length === 0) score += 20  // investidor sem preferência
     else if (investorTypes.includes(asset.type)) score += 35  // match perfeito

   Investidor sem nenhum critério preenchido pode atingir score 65+
   e aparecer como "match" para qualquer ativo.
   Na apresentação vai mostrar matches absurdos e sem sentido.

🟡 BUG 3 — camposEspecificos nos campos específicos do formulário
   Os campos CAR, CCIR, ITR, substância mineral etc são salvos em
   `camposEspecificos` no frontend mas não chegam ao banco pelo mesmo motivo do Bug 1.
   O formulário parece funcionar mas os dados são perdidos.

No arquivo `server/routes/matching.ts`, dentro da função de matching,
localize o bloco de score de tipo de ativo e substitua por completo:

SUBSTITUA:
```typescript
const investorTypes = (investor.assetTypes as string[]) || [];
if (investorTypes.length === 0) {
  score += 20;
  reasons.push("Investidor aceita todos os tipos de ativo");
} else if (investorTypes.includes(asset.type)) {
  score += 35;
  reasons.push(`Tipo "${asset.type}" está nas preferências do investidor`);
} else {
  penalties.push(`Tipo "${asset.type}" não está nas preferências do investidor`);
}
```

POR:
```typescript
const investorTypes = (investor.assetTypes as string[]) || [];

// Verifica se o investidor tem perfil minimamente preenchido
const perfilCompleto = investorTypes.length > 0 ||
  investor.ticketMin != null || investor.ticketMax != null ||
  ((investor.regionsOfInterest as string[]) || []).length > 0;

if (!perfilCompleto) {
  // Investidor sem nenhum critério → não gera match
  // Não adiciona pontos mas também não penaliza (skip completo)
  penalties.push("Perfil do investidor incompleto — sem critérios definidos");
} else if (investorTypes.length === 0) {
  // Investidor tem outros critérios mas aceita qualquer tipo
  score += 15;
  reasons.push("Investidor aceita qualquer tipo de ativo");
} else if (investorTypes.includes(asset.type)) {
  score += 40;
  reasons.push(`Tipo "${asset.type}" está nas preferências do investidor`);
} else {
  score -= 20; // penalidade forte por tipo errado
  penalties.push(`Tipo "${asset.type}" não está nas preferências do investidor`);
}
```

E logo antes do `if (score >= 40 && penalties.length <= 1)`, adicione:

```typescript
// Investidor sem perfil não deve gerar match
if (!perfilCompleto) continue;
```

```typescript
async getAssets(): Promise<Asset[]> {
```

POR:
```typescript
async getAssets(orgId?: number): Promise<Asset[]> {
  if (orgId) {
    return await db.select().from(assets)
      .where(eq(assets.orgId, orgId))
      .orderBy(desc(assets.createdAt));
  }
  return await db.select().from(assets).orderBy(desc(assets.createdAt));
}
```
