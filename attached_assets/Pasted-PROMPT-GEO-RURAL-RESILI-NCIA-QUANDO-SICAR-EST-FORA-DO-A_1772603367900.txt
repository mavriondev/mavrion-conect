PROMPT — GEO RURAL: RESILIÊNCIA QUANDO SICAR ESTÁ FORA DO AR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## Objetivo
Quando o SICAR (geoserver.car.gov.br) está fora do ar, o módulo Geo Rural
trava sem dar explicação ao usuário. Esta melhoria adiciona:
1. Detecção de status do SICAR em tempo real
2. Mensagem clara quando o serviço está fora
3. Modo de busca manual por código CAR como fallback
4. Link direto para o portal público do CAR

---

## Arquivo 1: `server/routes/geo-rural.ts`

Logo após a linha `const SICAR_WFS = "https://geoserver.car.gov.br/geoserver/sicar/wfs";`,
adicione:

```typescript
const CAR_PUBLIC_URL = "https://consultapublica.car.gov.br/publico/imoveis/index";
const CAR_DEMONSTRATIVO_URL = "https://www.car.gov.br/publico/imoveis/index";
```

Adicione esta nova rota ANTES da rota `/api/geo/fazendas`:

```typescript
// ── STATUS DO SICAR ──
app.get("/api/geo/sicar-status", async (req, res) => {
  if (!req.isAuthenticated()) return res.status(401).json({ message: "Não autenticado" });
  try {
    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), 8000);
    
    const testUrl = `${SICAR_WFS}?service=WFS&version=2.0.0&request=GetCapabilities`;
    const response = await fetch(testUrl, { 
      signal: controller.signal,
      method: "HEAD",
    }).catch(() => null);
    
    clearTimeout(timeout);
    
    const online = response?.ok || response?.status === 400;
    res.json({ 
      online,
      checkedAt: new Date().toISOString(),
      portalUrl: CAR_PUBLIC_URL,
    });
  } catch {
    res.json({ 
      online: false, 
      checkedAt: new Date().toISOString(),
      portalUrl: CAR_PUBLIC_URL,
    });
  }
});
```

Na rota `/api/geo/fazendas`, substitua o bloco `catch (err: any)` final pelo seguinte:

```typescript
    } catch (err: any) {
      console.error("Geo fazendas error:", err.message);

        portalUrl: CAR_PUBLIC_URL,
```

## Arquivo 2: `client/src/pages/geo-rural.tsx`

**Alteração 1** — Adicione este hook logo após os estados existentes no componente principal
(após as declarações de `useState`):

```typescript
const [sicarStatus, setSicarStatus] = useState<"checking" | "online" | "offline">("checking");
const [carManualCode, setCarManualCode] = useState("");
const [carManualLoading, setCarManualLoading] = useState(false);

useEffect(() => {
  apiRequest("GET", "/api/geo/sicar-status")
    .then(r => r.json())
    .then(data => setSicarStatus(data.online ? "online" : "offline"))
    .catch(() => setSicarStatus("offline"));
}, []);
```

## O que isso resolve

Antes: quando SICAR cai → tela trava → usuário não sabe o que fazer

Depois:
- Banner âmbar aparece quando SICAR está offline
- Campo para colar código CAR e abrir direto no portal público oficial
- Botão para ir ao portal completo do CAR
- Quando SICAR volta, banner some automaticamente
- A integração WFS original continua funcionando quando o serviço está no ar
